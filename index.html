What is HTML?

HTML stands for HyperText Markup Language. It is the standard markup language used to create and design web pages.

  • HyperText: Links between web pages.
  • Markup Language: Uses tags to define content.

Purpose of HTML

  • Defines the structure of web content.
  • Organizes text, images, links, forms, and multimedia.
  • Works with CSS and JavaScript.

HTML File Structure

<!DOCTYPE html>
<html>
  <head>
    <title>Page Title</title>
  </head>
  <body>
    <h1>Hello World</h1>
    <p>This is a paragraph.</p>
  </body>
</html>
Tag Purpose
<!DOCTYPE html> Declares HTML5 document type
<html> Root element
<head> Metadata and links
<title> Title in browser tab
<body> Main visible content

Common HTML Tags

Text Formatting

<h1> to <h6>, <p>, <br>, <strong>, <em>, <u>

Lists

<ul>, <ol>, <li>

Links & Images
<a href="url">Text</a>
<img src="image.jpg" alt="description" />
Tables
<table>
  <tr>
    <th>Header</th>
    <td>Data</td>
  </tr>
</table>
Forms
<form>
  <input type="text" name="username" />
  <input type="submit" />
</form>

Attributes in HTML

Attribute Description
id Unique identifier
class Class name
href URL for links
src Source for images
alt Alternative text
style Inline styles
title Tooltip text

Semantic HTML

Tags that describe their meaning clearly:

<header>, <footer>, <nav>, <article>, <section>, <main>

Helps with accessibility, SEO, and readability.

7. HTML5 Features

  • New semantic elements: <main>, <section>
  • Multimedia support: <audio>, <video>
  • Advanced form input types
  • Graphics with <canvas> and <svg>
  • Local storage capabilities

Best Practices

  • Use semantic tags
  • Indent and comment code
  • Use lowercase for tags and attributes
  • Separate CSS and JS from HTML
  • Validate HTML using W3C Validator

HTML with CSS and JavaScript

<!DOCTYPE html>
<html>
  <head>
    <link rel="stylesheet" href="style.css">
  </head>
  <body>
    <button onclick="alert('Hello!')">Click Me</button>
    <script src="script.js"></script>
  </body>
</html>

What is CSS?

Cascading Style Sheets (CSS) is a stylesheet language used to describe the presentation of a document written in HTML or XML (including XML dialects such as SVG, MathML or XHTML). CSS describes how elements should be rendered on screen, on paper, in speech, or on other media.

Purpose of CSS

  • Controls website look and feel.
  • Separates content (HTML) from design (CSS).
  • Improves reusability, maintainability, and consistency.

Types of CSS

Type Description Example
Inline Within HTML tags <p style="color:red;">Text</p>
Internal Within a <style> tag in head <style>p { color:red; }</style>
External Separate .css file <link rel="stylesheet" href="style.css">

CSS Syntax

selector {
  property: value;
}

Example:

h1 {
  color: blue;
  font-size: 24px;
}

CSS Selectors

  • * (Universal)
  • p (Element)
  • .class (Class)
  • #id (ID)
  • div p (Descendant)
  • a:hover (Pseudo-class)

Colors in CSS

  • Named: red, blue
  • Hex: #ff0000
  • RGB: rgb(255, 0, 0)
  • HSL: hsl(0, 100%, 50%)

CSS Units

  • Absolute: px, pt, cm
  • Relative: em, rem, %, vw, vh

Box Model

The CSS box model includes:

  • Content
  • Padding
  • Border
  • Margin

Common CSS Properties

Property Use
color Text color
background-color Element background
font-size Text size
margin / padding Space around and inside
border Element border
width / height Size control
display Layout behavior (block, flex, grid)

Positioning

  • static: Default
  • relative: Relative to normal position
  • absolute: Positioned relative to nearest ancestor
  • fixed: Fixed in viewport
  • sticky: Sticks during scroll

Flexbox

.container {
  display: flex;
  justify-content: center;
  align-items: center;
}

Main properties: flex-direction, justify-content, align-items, flex-wrap

CSS Grid

.container {
  display: grid;
  grid-template-columns: 1fr 1fr;
}

Main properties: grid-template-columns, grid-template-rows, grid-gap, grid-area

Media Queries (Responsive Design)

@media (max-width: 768px) {
  body {
    background-color: lightblue;
  }
}

Used to create layouts that adapt to screen sizes.

Cascading and Specificity

  • Last rule wins in conflicts.
  • Specificity hierarchy: Inline > ID > Class > Element

Best Practices

  • Use external stylesheets.
  • Use semantic and meaningful class names.
  • Organize code logically: reset, layout, components.
  • Avoid inline styles unless necessary.
  • Use developer tools and CSS validators.

üìò JavaScript - Notes

What is JavaScript?

JavaScript is a high-level, interpreted programming language used to make web pages interactive. It runs in the browser and is a core technology alongside HTML and CSS.

Why Use JavaScript?

  • To build dynamic and interactive web pages
  • To handle events (clicks, inputs, etc.)
  • To update HTML/CSS content on the fly
  • To validate forms before submission
  • To fetch data using APIs (AJAX, fetch, etc.)

Where to Use JavaScript?

  • Inline: <button onclick="alert('Hi!')">
  • Internal: Inside <script> tags
  • External: Separate .js file linked using <script src="script.js">

Basic Syntax

// Single-line comment
/* Multi-line comment */
let name = "John";
console.log("Hello " + name);

Variables

  • var ‚Äì old, function-scoped
  • let ‚Äì block-scoped
  • const ‚Äì block-scoped, cannot be reassigned
let age = 25;
const name = "Alice";

Data Types

  • String, Number, Boolean
  • Object, Array, Function
  • Undefined, Null, Symbol, BigInt
let isOnline = true;
let user = { name: "Alex", age: 30 };
let fruits = ["apple", "banana"];

Operators

  • Arithmetic: + - * / %
  • Assignment: = += -=
  • Comparison: == === != !== > <
  • Logical: && || !
let x = 5;
let y = 10;
console.log(x + y);

Control Structures

Conditional:
if (x > 10) {
  console.log("x is large");
} else {
  console.log("x is small");
}
Switch:
switch (day) {
  case "Monday":
    alert("Start of week");
    break;
  default:
    alert("Unknown day");
}
Loops:
for (let i = 0; i < 5; i++) {
  console.log(i);
}

while (x > 0) {
  x--;
}

Functions

function greet(name) {
  return "Hello " + name;
}

const sum = (a, b) => a + b;

Arrays and Objects

let colors = ["red", "green", "blue"];
colors.push("yellow");

let person = {
  name: "John",
  age: 28,
  greet: function() {
    return "Hi, " + this.name;
  }
};

DOM Manipulation

document.getElementById("demo").innerHTML = "Hello!";
document.querySelector(".box").style.color = "red";

Create an element and append:

let p = document.createElement("p");
p.textContent = "New paragraph";
document.body.appendChild(p);

Events

document.getElementById("btn").addEventListener("click", function() {
  alert("Button clicked!");
});

Asynchronous JS

Callback Example:
function loadData(callback) {
  setTimeout(() => {
    callback("Data loaded");
  }, 1000);
}
Promise Example:
let promise = new Promise((resolve, reject) => {
  resolve("Success");
});

promise.then(data => console.log(data));
Async/Await:
async function fetchData() {
  let res = await fetch("https://api.example.com/data");
  let data = await res.json();
  console.log(data);
}

ES6+ Features

  • let and const
  • Arrow functions: () => {}
  • Template literals: `Hello ${name}`
  • Destructuring: let { name } = person;
  • Spread/rest: [...arr], function(...args)
  • Modules: export/import

Best Practices

  • Use let and const instead of var
  • Keep code modular and DRY (Don‚Äôt Repeat Yourself)
  • Use meaningful variable/function names
  • Validate user inputs and handle errors
  • Comment where necessary
